# OSINT 开源情报分析方法论

> 本方法论整合自：库马尔《掌握开源情报的完整指南》、丁爸公众号系列、学术论文、国际标准
> 适用于商业竞争情报分析（而非军事/国家安全场景）

---

## 一、核心思维：信息 ≠ 情报

**丁爸方法论精华（思维层面）：**

> 信息被加工处理后，并且被用于支撑决策才是情报。
> 
> 情报的本质在于从众多的不确定性中找到相关性，并将各种细微的相关性串联起来，形成明确的判断，进而服务决策。

### 关键心法

1. **工具思维 vs 判断力思维**
   - ❌ 用工具而非判断力来衡量能力
   - ❌ 奖励速度而非思考
   - ❌ 将产出等同于洞察力
   - ✅ 当工具不再有用时，智慧就开始发挥作用

2. **自问五则**（情报分析师的自律）
   - 这里还缺少什么？
   - 什么地方说不通？
   - 我无意中做了哪些假设？
   - 什么因素会改变我的结论？我是否真的去寻找过这些因素？
   - 如果我错了怎么办？

3. **警惕"没有数据"陷阱**
   - 近年来最重要的情报都是在开放的互联网上获取的
   - 真正造成情报失误的并非信息获取渠道不足，而是情报分析师的思维和判断

4. **运营安全(OPSEC)即思维模式**
   - 运营安全应贯穿于信息收集和分析过程之中，而不是游离于其外
   - 人工智能改变了运营安全格局：当你把敏感信息粘贴到AI模型中，你是在把敏感的意图转移到别人的基础设施中

## 二、OSINT情报循环（标准流程）

### 学术标准六步循环

```
需求定义 → 信息搜集 → 处理整理 → 分析研判 → 报告输出 → 反馈优化
    ↑                                                      ↓
    ←←←←←←←←←←←←←← 持续迭代 ←←←←←←←←←←←←←←←←←←←←←←←←←←←←
```

### RIS螺旋情报循环（学术改进版）

来自 Reuser's Information Services (RIS) 的改进模型，将客户置于中心：

```
┌─────────────────────────────────────────────────────┐
│                    客户（中心）                       │
│                         ↑                           │
│    ┌────────────────────┼────────────────────┐     │
│    │                    │                    │     │
│    ↓                    ↓                    ↓     │
│ 准备循环          报告循环            情报生产循环      │
│  -需求分析        -搜索采集          -综合分析        │
│  -信源分析        -整理索引          -情报报告        │
│  -收集规划        -信息报告          -分发传播        │
│    │                    │                    │     │
│    └────────────────────┼────────────────────┘     │
│                         ↓                           │
│                   知识管理（沉淀复用）                │
└─────────────────────────────────────────────────────┘
```

### Step 1：需求定义
- 明确情报目标（要回答什么问题？）
- 确定调查范围（对象、时间、地域）
- 选择数据源和工具
- 风险评估与合规检查

### Step 2：信息搜集

| 收集方式 | 说明 | 典型场景 |
|----------|------|----------|
| **被动侦察** | 不直接与目标通信，通过第三方数据库获取 | 搜索引擎、历史档案 |
| **半被动侦察** | 轻量通信伪装成普通流量 | API查询、公共数据源 |
| **主动侦察** | 直接查询目标系统 | 公开表单查询、人脉询问 |

### Step 3：处理整理
- **清洗**：去除无关信息
- **去重**：合并重复内容
- **提取**：结构化关键信息（人名、公司名、时间、地点、数据）
- **翻译**：处理外语内容

### Step 4：分析研判
- **交叉验证**：多源信息一致性
- **关联分析**：建立时间线、关系图
- **模式识别**：发现规律和异常
- **逻辑推理**：排除矛盾、得出结论

### Step 5：报告输出
- 明确结论和依据
- 标注可信度等级
- 提供证据链
- 给出行动建议

### Step 6：反馈优化
- 收集用户反馈
- 调整情报需求
- 优化分析方法
- 知识沉淀复用

## 三、信源分级体系

### NATO Admiralty Code（国际标准）

| 来源可靠性 | 说明 | 信息可信度 | 说明 |
|------------|------|------------|------|
| **A - 完全可靠** | 已证实可靠 | **1 - 已确认** | 被其他独立来源证实 |
| **B - 通常可靠** | 历史表现良好 | **2 - 几乎确定** | 无理由怀疑 |
| **C - 相当可靠** | 曾有良好表现 | **3 - 可能为真** | 过去正确率高 |
| **D - 通常不可靠** | 需交叉验证 | **4 - 可疑** | 真实性不确定 |
| **E - 不可靠** | 曾提供虚假信息 | **5 - 难以置信** | 不太可能是真的 |
| **F - 可靠性无法判断** | 未知来源 | **6 - 真实性无法判断** | 无法验证 |

### 商业情报信源分级（适用版）

| 等级 | 定义 | 信源类型 | 采信程度 |
|------|------|----------|----------|
| **A** | 多源交叉验证 | 专利+官网+学术三方一致 | 直接采信，高置信度 |
| **B** | 官方单源 | 厂商官网、财报、专利公开 | 可采信，标注单源 |
| **C** | 权威媒体 | 有采编资质的行业媒体 | 需追溯原始来源后采信 |
| **D** | 待确认 | 社交媒体线索、自媒体、无来源 | 仅作线索，需验证 |

### 信源类型排序（从高到低）

1. **专利文献**：CNIPA、USPTO、EPO、WIPO
2. **官方发布**：厂商官网、财报、投资者关系材料、白皮书
3. **学术来源**：IEEE、Elsevier、CNKI等数据库论文
4. **权威媒体**：有采编资质的行业媒体
5. **展会/会议**：官方展位发布、技术演讲
6. **公共记录**：工商注册、法院文书、招投标公告
7. **社交媒体**：微博、公众号、论坛（仅作线索源）

### 文献加工深度分级

| 等级 | 类型 | 说明 |
|------|------|------|
| **零次文献** | 口头交谈、未公开资料 | 直接但未经正式记录 |
| **一次文献** | 原始研究、论文、专利 | 创新性、实用性、学术性 |
| **二次文献** | 目录、索引、文摘 | 检索工具，非原始内容 |
| **三次文献** | 综述、评论、报告 | 综合分析，高引用价值 |

## 四、R2C2删选原则

对每条信息都要过这四道筛子：

| 维度 | 问题 | 判断标准 |
|------|------|----------|
| **R**elevant（相关） | 是否相关？ | 与目标主题、时间范围、地域范围是否匹配 |
| **R**eliable（可靠） | 来源是否可靠？ | 官方>学术>权威媒体>自媒体>匿名来源 |
| **C**redible（可信） | 内容是否可信？ | 逻辑是否自洽、有无矛盾、是否合理 |
| **C**orroborated（佐证） | 是否有佐证？ | 是否有≥2个独立来源印证 |

**操作口诀**：
> 相关吗？来源可靠吗？内容可信吗？有其他佐证吗？

## 五、Three Confirms验证规则

| 确认数量 | 等级 | 说明 |
|----------|------|------|
| 1个来源 | 好奇 | 仅作为线索追踪 |
| 2个来源 | 前景 | 值得深入调查 |
| 3个独立来源 | 工作级事实 | 可纳入情报结论 |

**注意**：Reuters引用BBC，BBC引用Reuters = 单一来源循环验证（circular reporting）

**真正独立的证据类型**：
- 官方注册文件（文档痕迹）
- 视频+地理定位（视觉痕迹）
- 网络流量/日志（技术痕迹）

## 六、Analysis of Competing Hypotheses (ACH)

CIA开发的认知偏差对抗方法：

### 操作流程

1. **列出假设**：H1（主要版本）、H2（替代方案）、H3（偶然/错误）
2. **列出证据**：所有相关证据E1、E2、E3...
3. **构建矩阵**：验证每个证据是否支持每个假设
4. **评估一致性**：是否与多个假设都兼容？（兼容性过强=证据效力低）
5. **得出结论**

### 示例

```
证据E = "照片中的人看起来像X"
- 支持H1（X有罪）？是（但巧合率1:1000）
- 支持H2（随机路人）？是（巧合率1:1000）
- 支持H3（照片是旧的）？是
结论：该照片不能作为有罪证据，因为它与所有假设都兼容
```

## 七、信息验证方法论

### 1. 时间线验证（Temporal Analysis）

| 层级 | 证据类型 | 可靠性 |
|------|----------|--------|
| UI层级 | "2小时前发布" | 不稳定 |
| HTML meta | `<meta itemprop="uploadDate">` | 较好 |
| JSON响应 | `"created_at": "ISO8601"` | 金标准 |

### 2. 元数据验证（Metadata Verification）

| 对象 | 工具 | 提取信息 |
|------|------|----------|
| 图片 | ExifTool | GPS、相机型号、软件痕迹 |
| 文档 | FOCA | 用户名、路径、软件版本 |
| 视频 | MediaInfo | 编码信息、时间戳 |

### 3. 反向图片搜索

| 工具 | 用途 |
|------|------|
| Google Images | 通用反向搜索 |
| TinEye | 精确匹配查找 |
| Yandex | 面部识别更强 |
| Bing Visual Search | 综合搜索 |

### 4. 网站历史验证

- **Wayback Machine**：查看历史页面版本
- **Compare功能**：对比代码变化
- **发现**：已删除内容、隐性修改

## 八、常见陷阱与应对

### 伪信源识别

| 陷阱类型 | 特征 | 应对方法 |
|----------|------|----------|
| 仿冒正规渠道 | 域名怪异、页面模仿 | 检查域名、查找背景口碑 |
| 断章取义 | 二手引用、只言片语 | 追溯原文全文 |
| 旧闻翻炒 | 老照片/视频当新闻 | 反向图片搜索、时间线核对 |
| 数据误导 | 图表篡改、选择性呈现 | 查证原始数据来源 |
| 循环引用 | 多源引用同一信息 | 追溯真正独立来源 |

### 深度伪造检测

| 类型 | 识别特征 |
|------|----------|
| AI生成文本 | 语义不连贯、语法过于完美 |
| AI生成图片 | 面部比例失调、光影异常 |
| 换脸视频 | 动作僵硬、声音不匹配 |
| 设备指纹 | 不同"独立"照片EXIF显示同一相机 |

### 常见错误

- ❌ 盲信单一来源
- ❌ 忽视时间线矛盾
- ❌ 不追溯原始出处
- ❌ 被利益相关方误导
- ❌ 混淆"实验室数据"与"量产数据"

## 九、模板体系

### 情报报告模板

```markdown
# [主题]情报报告
**日期**：YYYY-MM-DD  
**分析师**：[姓名]

## 一、已证实情报（A级）
[高置信度信息，带证据链]

## 二、待验证线索（D级）
[存疑信息，需进一步验证]

## 三、分析结论
[基于证据的判断]

## 四、可信度标注
[每条结论的可信度来源]
```

### 企业调查模板

| 模块 | 内容 |
|------|------|
| 基础信息 | 公司名称、注册信息、股东结构 |
| 财务状况 | 财报摘要、融资历史、估值 |
| 技术实力 | 专利布局、研发投入、技术团队 |
| 市场动态 | 产品发布、合作伙伴、竞争动作 |
| 风险提示 | 法律诉讼、监管处罚、舆论危机 |

## 十、AI时代的OSINT

### AI辅助能力

| 任务 | AI优势 | 注意事项 |
|------|--------|----------|
| 数据收集 | 快速、全面 | 可能引入噪声 |
| 信息关联 | 模式识别 | 需人工验证关联合理性 |
| 摘要生成 | 高效 | 可能产生幻觉 |
| 实体提取 | 自动化 | 需人工校正 |
| 图像分析 | 快速识别 | 深度伪造难以检测 |

### 关键原则

> 人工智能并没有取代分析师（它取代的是分析的假象）。

**AI能做**：
- 告诉你正在发生什么
- 提供素材和线索
- 关联性分析
- 叙述性总结

**AI不能做**：
- 解释为什么它很重要
- 承担犯错的后果
- 回答"还缺少什么"这类问题
- 进行真正的假设检验
